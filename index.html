RoboChallenge Track Guide RoboChallenge 赛道指南

Evaluate multi-task embodied intelligence models on real physical hardware via RoboChallenge. 通过 RoboChallenge 平台在真实物理硬件上评测多任务具身智能模型。

Overview 总体介绍

This track focuses on real-world physical robot manipulation and adopts a multi-task strategy. Participants are required to use a single model to complete all designated tasks. There are no restrictions on the model type; both World Models and VLA models are welcome. Participants will use the RoboChallenge platform to submit evaluation tasks and execute models on real physical hardware.

本赛道专注于真实物理世界的机器人操作，并采用多任务（multi-tasks）赛制策略。参赛者需要使用单个模型来完成所有指定的测试任务。赛道对参赛模型类型不做限制，世界模型（World Model）与 VLA 模型均可参赛。参赛者将使用 RoboChallenge 平台提交评测任务，并在真实物理硬件上执行模型推理结果。

The official Table30 for CVPR dataset and the dedicated track leaderboard are currently being prepared. They will be released shortly. In the meantime, participants can complete the registration, environment setup, and local Mock testing to prepare their pipelines.

Step-by-step Guidance 参赛步骤

Step 1. Registration & Setup 步骤 1：账号注册与社群接入

⚠️ Note: Each team can only apply for ONE account. ⚠️ 注意：每支队伍只能申请一个账号。

  • Send a registration application email to join-robochallenge@robochallenge.ai using the template below:
  • 请使用以下模板向赛事官方邮箱 join-robochallenge@robochallenge.ai 发送注册申请邮件：
Subject: User Account Application Request - CVPR 2026 Workshop Competition Dear RoboChallenge Team, I would like to apply for a user account for RoboChallenge Platform. My details are as follows: Team Name: [Your Team Name] Nationality: [Your Nationality] Institute/Organization: [Your Institute / Organization] Department: [Your Department] Position/Title: [Your Position] Email Address: [Your Email] Phone Number: [Your Phone Number] Purpose of Application: Participating in the CVPR 2026 Workshop Competition
  • Once approved, you will receive a User Token via email.
  • Crucial: Scan the QR code in the reply email to join the official WeChat support group for real-time assistance and to schedule your on-device tests later.
  • 审核通过后，官方将发放包含专属令牌 (Token) 的确认邮件。
  • 必做事项：通过邮件中的二维码加入官方微信群，以便后续沟通和预约真机测试时间。

Step 2. Environment & Local Mock Testing 步骤 2：环境搭建与本地 Mock 调试

Before real-robot evaluation, you must adapt your model using the official inference repository.

在进行真机测试前，需基于官方推理仓库完成环境搭建与代码适配。

1. Clone Repository: 1. 克隆仓库：
git clone https://github.com/RoboChallenge/RoboChallengeInference.git
pip install -r requirements.txt
2. Start Mock Server: 2. 启动本地 Mock 服务：

Edit mock_settings.py to configure your target robot tag (e.g., 'aloha', 'arx5', 'ur5', 'franka'). Then run: 编辑 mock_settings.py 设置目标机器人（如 'aloha', 'arx5', 'ur5', 'franka'）。随后运行：

python3 mock_robot_server.py
3. Execute Test Script: 3. 执行调试脚本：

Run test.py to verify that your custom policy properly interacts with the Mock interface. 运行 test.py 验证推理逻辑能否成功生成动作指令并下发至 Mock 机器人。

Step 3. Online Submission & Real Robot Execution 步骤 3：线上提交与真机执行

  • Log in and click Eval Your Policy. Since this track utilizes a multi-task strategy, select Multitask as your Model Type and select the tasks you wish to evaluate.
  • After submission, go to My Submissions to get your Run ID.
  • Negotiate a testing time slot with the official team via the official WeChat group.
  • During your slot, run the formal script locally to interact with the physical device via HTTP:
  • 登录后点击 Eval Your Policy 按钮。由于赛制要求，请在模型类型中选择 Multitask，并勾选你需要评测的任务。
  • 提交后在 My Submissions 页面获取你的 Run ID。
  • 通过官方微信群与官方协商本条 Run ID 的测试时间。
  • 在指定时间段内，本地运行正式推理脚本，程序将与真实物理设备交互：
python3 demo.py --user_token {Your Token} --run_id {Your Run ID} --checkpoint {Model Path}
